Tema 11: Archivos de Texto

Inicio

Introducción

Los archivos de texto son una de las formas más básicas y ampliamente utilizadas para almacenar información de manera permanente en los sistemas informáticos. A diferencia de los archivos binarios, los archivos de texto contienen datos codificados en caracteres legibles por humanos, lo que facilita su creación, edición y análisis.

Desde simples configuraciones hasta registros complejos, los archivos de texto juegan un papel esencial en el desarrollo de aplicaciones, la administración de sistemas y la transferencia de datos entre plataformas. En este tema, exploraremos a fondo cómo funcionan, cómo se manipulan en distintos lenguajes de programación y las mejores prácticas para su uso.

¿Qué es un archivo de texto?

Un archivo de texto es un documento que contiene una secuencia de caracteres codificados en formatos como ASCII o UTF-8. Estos archivos pueden tener cualquier extensión, aunque las más comunes son .txt, .log, .csv, entre otros.

Son especialmente útiles para almacenar:

  • Notas o registros simples
  • Archivos de configuración (ej. INI, CFG, YAML)
  • Datos separados por delimitadores (CSV, TSV)
  • Documentación, scripts o incluso código fuente

Operaciones Básicas con Archivos de Texto

Las operaciones más comunes que se realizan con archivos de texto incluyen:

  • Abrir y cerrar archivos
  • Leer contenido (línea por línea o todo el archivo)
  • Escribir o sobreescribir contenido
  • Agregar contenido sin borrar el anterior
  • Eliminar o renombrar archivos

Ejemplo en Python

# Escribir en un archivo
with open("notas.txt", "w") as archivo:
    archivo.write("Primera línea de texto\n")
    archivo.write("Segunda línea")

# Leer el archivo
with open("notas.txt", "r") as archivo:
    for linea in archivo:
        print(linea.strip())

Modos de Apertura

La manera en que abrimos un archivo determina cómo interactuamos con él. Los modos de apertura más comunes son:

Modo Descripción
"r" Abrir para leer. Falla si no existe.
"w" Abrir para escribir. Sobrescribe si ya existe.
"a" Abrir para agregar al final del archivo.
"r+" Abrir para leer y escribir.

Lectura y Escritura en otros lenguajes

Java

import java.io.*;

public class LeerArchivo {
    public static void main(String[] args) {
        try {
            BufferedReader lector = new BufferedReader(new FileReader("datos.txt"));
            String linea;
            while ((linea = lector.readLine()) != null) {
                System.out.println(linea);
            }
            lector.close();
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

C++

#include <fstream>
#include <iostream>
using namespace std;

int main() {
    ofstream archivo("archivo.txt");
    archivo << "Hola, mundo\n";
    archivo.close();

    string linea;
    ifstream lectura("archivo.txt");
    while (getline(lectura, linea)) {
        cout << linea << endl;
    }
    lectura.close();
    return 0;
}

Ventajas de los Archivos de Texto

  • Compatibilidad: Son legibles por cualquier sistema operativo.
  • Portabilidad: Fácil de compartir y transferir.
  • Simplicidad: No requieren estructuras complejas.
  • Edición Manual: Pueden abrirse en cualquier editor de texto.

Limitaciones

  • No adecuados para datos binarios o multimedia.
  • Carecen de mecanismos de seguridad o validación.
  • Poca eficiencia en estructuras complejas.
  • Mayor tamaño que formatos comprimidos o binarios.

Aplicaciones Reales

  • Sistemas de logging (archivos de registros)
  • Exportación e importación de datos entre sistemas
  • Configuraciones para videojuegos, servidores y software
  • Almacenamiento de scripts y código fuente

Buenas Prácticas

  • Utilizar codificación UTF-8 para compatibilidad internacional.
  • Verificar siempre la existencia del archivo antes de abrirlo.
  • Cerrar correctamente los archivos después de su uso.
  • Evitar sobrescribir accidentalmente archivos importantes.

Conclusión

Los archivos de texto son una herramienta sencilla pero poderosa en el mundo de la programación. Permiten registrar información de forma directa, sin requerir bases de datos u otras estructuras complejas. Aunque tienen limitaciones frente a otros formatos más sofisticados, su facilidad de uso, portabilidad y compatibilidad los convierten en una opción valiosa en múltiples contextos. Dominar su manipulación es fundamental para cualquier desarrollador de software.
